# Agentic Engineering 核心模式参考

基于 Simon Willison（Django 框架联合创始人）的 Agentic Engineering Patterns，提供 AI 编码代理的核心工程原则。

## When to Activate
- 使用 AI 编码代理开发时
- 需要理解 LLM 协作最佳实践时
- 规划 AI 辅助开发流程时

## 核心理念：Writing Code is Cheap Now

### 代码成本革命
- 代码生成成本已接近零
- 高质量代码仍有成本
- 需要建立新的工程习惯

### 高质量代码定义
1. 代码正确工作，无 Bug
2. 我们知道代码工作正常（已验证）
3. 解决正确的问题
4. 优雅处理错误情况
5. 简洁最小，易于理解维护
6. 有测试保护
7. 文档完善且同步更新
8. 设计支持未来变更（YAGNI 平衡）

## 十大关键策略

### 策略 1：设定合理期望
- LLM 本质是"基于概率的自动补全"
- 擅长模式匹配而非逻辑推理
- 适合：生成代码片段、修复简单 Bug、撰写测试用例
- 不适合：端到端开发、复杂算法设计

### 策略 2：训练截止日期应对
- 优先选择稳定库（如 requests 而非 httpx 新版本）
- 主动提供新库文档作为上下文
- 注意模型知识截止日期

### 策略 3：上下文管理艺术
- 对话重置：复杂任务适时清空对话
- 上下文注入：先提供相似代码再要求修改
- 分步引导：先生成简单实现再逐步添加功能

### 策略 4：技术调研智能化
- 开放性问题："Rust 中有哪些异步 HTTP 客户端？"
- 对比分析："比较 Pandas 与 Polars 性能"
- 注意模型可能遗漏冷门库

### 策略 5：精确任务指令
- 定义清晰的函数签名和需求
- 通过注释指定代码风格（Black 格式化、类型注解）
- 示例：async def fetch_data(url: str, retries: int=3) -> list[dict]

### 策略 6：测试不可替代性
- 模型可能忽略边缘情况（网络超时、空数据）
- 可能存在安全漏洞（SQL 注入）
- 要求模型生成测试用例
- 结合覆盖率工具验证

### 策略 7：迭代式对话
- 定位问题：运行代码，记录错误信息
- 精准反馈："第30行的parse_response未处理 JSONDecodeError"
- 要求重构："将重复代码提取为validate_response函数"
- 要求解释验证理解

### 策略 8：沙箱化执行环境
- ChatGPT Code Interpreter：安全执行 Python
- Claude Artifacts：实时预览 Web 应用
- 避免在生产环境直接运行未审核代码

### 策略 9：Vibe Coding 实践
- 快速实验：修改提示观察输出变化
- 知识沉淀：积累可复用 Prompt 模板
- 案例参考：构建小型工具积累提示模式

### 策略 10：专业知识放大效应
- 熟悉 React 的开发者能更高效引导模型
- 将领域经验转化为精准的 Prompt 约束
- "遵循 RESTful 规范"等约束性指令

## 工程习惯建议

### 新习惯建立
- 任何"不值得花时间"的想法都可以尝试
- 异步代理会话中，最坏情况只是浪费 tokens
- 10分钟后检查结果，不满意就放弃

### 风险意识
- 始终保持对最终决策权的掌控
- 系统设计、安全性、关键逻辑需人工把关
- 生成代码必须经过审查和测试

## Quick Reference

| 策略 | 关键点 | 应用场景 |
|------|--------|----------|
| 设定期望 | 概率补全，非逻辑推理 | 功能规划 |
| 截止日期 | 选稳定库，提供文档 | 技术选型 |
| 上下文管理 | 重置对话，注入示例 | 复杂任务 |
| 技术调研 | 开放问题，对比分析 | 方案评估 |
| 精确指令 | 清晰签名，风格注释 | 代码生成 |
| 测试必须 | 边缘情况，安全检查 | 质量保证 |
| 迭代对话 | 定位问题，精准反馈 | 错误修正 |
| 沙箱执行 | 安全环境，避免生产 | 代码验证 |
| Vibe Coding | 快速实验，模板积累 | 原型开发 |
| 知识放大 | 专业引导，约束转化 | 效率提升 |
