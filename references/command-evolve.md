# evolve 命令参考

## 命令概述
`evolve` 命令用于将相关的本能（instincts）聚类成技能（skills），帮助系统自动组织和优化学习成果，提高知识的结构化程度和可复用性。

## 命令语法
```bash
fullstack-dev-skills evolve [选项]
```

## 参数说明
| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `--threshold` | 数字 | 否 | 聚类阈值，控制聚类的紧密程度，范围 0-1，默认 0.7 |
| `--min-instincts` | 数字 | 否 | 每个技能的最小本能数量，默认 3 |
| `--filter` | 字符串 | 否 | 按类别或关键词过滤要处理的本能 |
| `--min-confidence` | 数字 | 否 | 最小置信度阈值，低于此值的本能将被跳过，默认 0.5 |
| `--dry-run` | 布尔值 | 否 | 模拟执行，不实际创建技能，默认 `false` |
| `--output` | 字符串 | 否 | 输出格式，支持 `json` 和 `text`，默认 `text` |

## 工作原理
1. **数据收集**：从本地存储中读取本能数据
2. **过滤处理**：根据过滤参数和置信度阈值筛选本能
3. **相似度计算**：计算本能之间的语义相似度
4. **聚类分析**：使用聚类算法将相似的本能分组
5. **技能创建**：为每个聚类创建一个新技能
6. **结果汇总**：生成进化结果报告

## 常见问题

### Q: 什么是本能进化？
A: 本能进化是指将系统学习到的零散本能（instincts）自动组织成结构化的技能（skills）的过程。通过聚类算法，系统可以识别相关的本能并将它们分组，形成更有意义和可复用的知识单元。

### Q: 进化过程是如何工作的？
A: 进化过程包括以下步骤：
1. 收集和过滤本能数据
2. 计算本能之间的语义相似度
3. 使用聚类算法将相似的本能分组
4. 为每个聚类创建一个新技能
5. 生成进化结果报告

### Q: 如何选择合适的聚类阈值？
A: 选择聚类阈值时考虑以下因素：
- **高阈值**（0.8-1.0）：生成更少但更紧密相关的技能
- **中等阈值**（0.6-0.8）：生成适中数量的技能，平衡相关性和覆盖范围
- **低阈值**（0.4-0.6）：生成更多但相关性较低的技能

### Q: 进化后的技能与手动创建的技能有什么区别？
A: 进化后的技能与手动创建的技能的区别：
- **进化技能**：自动生成，基于系统学习的本能，反映实际的学习成果
- **手动技能**：用户创建，基于特定需求，更加目标导向

两者可以互补使用，进化技能提供自动组织的知识结构，手动技能提供特定领域的专业知识。

### Q: 进化过程会修改现有的本能或技能吗？
A: 进化过程不会修改现有的本能或技能，而是基于现有的本能创建新的技能。您可以保留或删除进化生成的技能，不会影响原始的本能数据。

### Q: 如何优化进化结果？
A: 优化进化结果的方法包括：
- 调整 `--threshold` 参数，找到适合您需求的聚类紧密程度
- 使用 `--filter` 参数，分别处理不同类别的本能
- 设置适当的 `--min-confidence` 阈值，确保只处理高质量的本能
- 尝试不同的参数组合，比较进化结果

### Q: 进化生成的技能如何使用？
A: 进化生成的技能可以通过以下方式使用：
- 在任务执行中自动应用，提供相关的知识和建议
- 作为手动参考，了解系统的学习成果
- 手动编辑和调整，使其更符合您的需求
- 与其他技能组合使用，形成更完整的知识体系

### Q: 定期运行进化命令有什么好处？
A: 定期运行进化命令的好处包括：
- 保持技能的更新和优化
- 适应新学习的本能
- 发现知识之间的新联系
- 提高系统的整体性能和准确性
- 减少本能的冗余和重复

### Q: 进化失败的原因有哪些？
A: 进化失败的常见原因包括：
- 系统资源不足
- 本能数据不足或质量太低
- 参数设置不当（如阈值过高）
- 系统内部错误

如果进化失败，建议检查系统资源和参数设置，或尝试处理更小范围的本能。
