# instinct-import 命令参考

## 命令概述
`instinct-import` 命令用于从外部文件导入本能（instincts），帮助用户快速积累知识和技能，或在不同系统之间迁移学习成果。

## 命令语法
```bash
fullstack-dev-skills instinct-import [选项] --file <文件路径>
```

## 参数说明
| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `--file` | 字符串 | 是 | 本能文件的路径，支持 JSON 格式 |
| `--merge` | 布尔值 | 否 | 是否合并到现有本能，默认 `true` |
| `--override` | 布尔值 | 否 | 是否覆盖相同 ID 的本能，默认 `false` |
| `--filter` | 字符串 | 否 | 按类别或关键词过滤要导入的本能 |
| `--min-confidence` | 数字 | 否 | 最小置信度阈值，低于此值的本能将被跳过，默认 0 |
| `--output` | 字符串 | 否 | 输出格式，支持 `json` 和 `text`，默认 `text` |

## 工作原理
1. **文件读取**：读取指定路径的本能文件
2. **数据验证**：验证文件格式和本能数据的有效性
3. **过滤处理**：根据过滤参数和置信度阈值筛选本能
4. **冲突检测**：检测与现有本能的冲突
5. **数据导入**：将筛选后的本能导入到系统中
6. **结果汇总**：生成导入结果报告

## 本能文件格式

本能文件必须是有效的 JSON 文件，格式如下：

```json
{
  "version": "1.0",
  "instincts": [
    {
      "id": "instinct-123",
      "category": "代码生成",
      "subcategory": "React",
      "description": "生成符合 React 最佳实践的功能组件",
      "confidence": 0.98,
      "examples": [
        "包含 useState、useEffect 等 hooks 的使用",
        "符合 React 组件命名规范"
      ],
      "context": {
        "languages": ["javascript", "typescript"],
        "frameworks": ["react"]
      },
      "created_at": "2024-01-15T14:30:22Z",
      "last_used": "2024-01-15T14:30:22Z",
      "usage_count": 10
    },
    // 更多本能...
  ]
}
```

### 必需字段
- `id`：本能的唯一标识符
- `category`：本能的类别
- `description`：本能的描述
- `confidence`：本能的置信度分数（0-1）

### 可选字段
- `subcategory`：本能的子类别
- `examples`：本能的使用示例
- `context`：本能的上下文信息
- `created_at`：本能的创建时间
- `last_used`：本能的最后使用时间
- `usage_count`：本能的使用次数

## 常见问题

### Q: 什么类型的本能文件可以导入？
A: 您可以导入以下类型的本能文件：
- 使用 `instinct-export` 命令导出的文件
- 手动创建的符合格式要求的 JSON 文件
- 从其他系统或用户处获取的本能文件

### Q: 导入本能会影响现有的学习成果吗？
A: 默认情况下，导入的本能不能会覆盖现有本能，而是会被合并到系统中。您可以使用 `--override` 参数来强制覆盖相同 ID 的本能。

### Q: 如何确保导入的本能质量？
A: 您可以使用以下参数来控制导入的本能质量：
- `--min-confidence`：设置最小置信度阈值
- `--filter`：只导入特定类别的本能
- `--merge`：控制是否合并到现有本能

### Q: 导入大量本能会影响系统性能吗？
A: 导入大量本能可能会暂时影响系统性能，但导入完成后，系统会对本能进行优化存储和索引，不会长期影响性能。建议分批导入非常大的本能文件。

### Q: 导入失败的原因有哪些？
A: 导入失败的常见原因包括：
- 文件格式无效或损坏
- 本能数据缺少必要字段
- 文件权限问题
- 磁盘空间不足

### Q: 导入的本能与现有本能冲突怎么办？
A: 当导入的本能与现有本能冲突时，默认情况下会跳过这些本能。您可以：
- 使用 `--override true` 强制覆盖现有本能
- 手动编辑本能文件，修改冲突的 ID
- 接受跳过的结果，保留现有本能

### Q: 如何验证导入的本能是否成功？
A: 您可以使用 `instinct-status` 命令查看导入后的本能状态，确认导入的本能是否存在以及它们的置信度分数。
