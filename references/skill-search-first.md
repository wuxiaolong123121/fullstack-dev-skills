# 搜索优先模式 (Search-First Mode)

## 概述

搜索优先模式是一种开发方法论，强调在编写代码之前先搜索和理解现有代码库。通过先搜索、后编码的方式，避免重复造轮子，保持代码一致性，提高开发效率。

## 核心原则

### 1. 先搜索，后编码
在编写任何新代码之前，先搜索代码库中是否已存在类似实现。

### 2. 理解上下文
理解现有代码的结构、模式和约定，确保新代码与现有代码风格一致。

### 3. 复用优先
优先复用现有代码，避免重复实现相同功能。

### 4. 文档驱动
搜索时同时查看文档和注释，理解代码的设计意图。

## 搜索策略

### 1. 关键词搜索
```
搜索模式: function|class|interface|const|let|var [关键词]
示例: "function validate" | "class User" | "interface Config"
```

### 2. 功能搜索
```
搜索模式: [功能描述关键词]
示例: "authentication" | "validation" | "pagination"
```

### 3. 模式搜索
```
搜索模式: [设计模式名称]
示例: "factory" | "singleton" | "observer" | "strategy"
```

### 4. API 搜索
```
搜索模式: [API 端点或方法名]
示例: "/api/users" | "getUserById" | "createOrder"
```

## 搜索工具

### 代码搜索
```bash
# 搜索函数定义
grep -r "function.*validate" src/

# 搜索类定义
grep -r "class.*Service" src/

# 搜索接口定义
grep -r "interface.*Config" src/

# 搜索导入语句
grep -r "from.*utils" src/
```

### 文件搜索
```bash
# 按文件名搜索
find . -name "*service*.ts"

# 按文件类型搜索
find . -name "*.test.ts"

# 按目录搜索
find . -type d -name "components"
```

### 高级搜索
```bash
# 正则表达式搜索
grep -rE "(async|function)\s+\w+Handler" src/

# 多条件搜索
grep -r "export" src/ | grep "interface"

# 排除目录搜索
grep -r "TODO" src/ --exclude-dir=node_modules
```

## 最佳实践

### 1. 搜索时机
- 开始新功能开发前
- 遇到技术问题时
- 重构代码前
- 学习代码库时

### 2. 搜索范围
- 优先搜索核心模块
- 搜索测试文件了解用法
- 搜索配置文件了解配置
- 搜索文档了解设计

### 3. 搜索深度
- 第一层: 精确匹配搜索
- 第二层: 模糊匹配搜索
- 第三层: 相关概念搜索
- 第四层: 外部资源搜索

### 4. 结果处理
- 评估代码质量和可维护性
- 检查测试覆盖率
- 确认是否有依赖问题
- 考虑是否需要重构

## 使用场景

### 场景 1: 新功能开发
```
任务: 实现用户头像上传功能

搜索步骤:
1. 搜索 "upload" - 找到现有上传工具
2. 搜索 "image" - 找到图片处理逻辑
3. 搜索 "avatar" - 确认是否有现有实现
4. 搜索 "storage" - 了解存储配置

结果: 复用现有上传工具，扩展支持图片处理
```

### 场景 2: Bug 修复
```
任务: 修复用户登录失败问题

搜索步骤:
1. 搜索 "login" - 找到登录相关代码
2. 搜索 "auth" - 找到认证逻辑
3. 搜索 "error" - 找到错误处理
4. 搜索 "session" - 了解会话管理

结果: 定位问题，参考现有错误处理模式修复
```

### 场景 3: 代码重构
```
任务: 重构数据验证逻辑

搜索步骤:
1. 搜索 "validate" - 找到所有验证代码
2. 搜索 "schema" - 找到数据模型定义
3. 搜索 "error" - 找到错误处理方式
4. 搜索 "test" - 找到相关测试

结果: 统一验证模式，提取公共验证工具
```
