# 线性代码演示模式参考

生成结构化的代码演示文档，用于理解现有代码库或学习新技术栈。

## When to Activate
- 需要理解现有代码库时
- 学习新技术栈时
- 需要为项目生成文档时
- Vibe Coding 后需要理解生成代码时

## 核心理念

### 为什么需要代码演示
- 快速理解复杂代码库
- 学习新技术和框架
- 记录代码架构和设计决策
- 帮助新成员上手项目

### 线性演示原则
- 按逻辑顺序组织内容
- 使用工具输出代码片段（避免手动复制）
- 提供清晰的上下文说明
- 包含实际运行示例

## Showboat 工具使用指南

### 工具介绍
Showboat 是一个帮助编码代理编写演示文档的工具，确保代码片段通过工具命令输出而非手动复制。

### 核心命令

```bash
# 添加注释到文档
showboat note "这是对代码的解释说明"

# 执行命令并添加输出到文档
showboat exec cat src/main.py

# 查看帮助
showboat --help
```

### 使用示例

```javascript
/**
 * Showboat 使用示例
 */
const showboatWorkflow = {
    step1: {
        command: 'showboat note "## 项目结构概览"',
        description: '添加章节标题'
    },
    step2: {
        command: 'showboat exec find . -name "*.py" | head -10',
        description: '列出项目文件'
    },
    step3: {
        command: 'showboat note "### 主入口文件"',
        description: '添加子章节'
    },
    step4: {
        command: 'showboat exec cat src/main.py',
        description: '展示主入口代码'
    },
    step5: {
        command: 'showboat note "这段代码实现了..."',
        description: '添加代码解释'
    }
};
```

## 代码片段输出最佳实践

### 使用工具而非手动复制

```javascript
/**
 * 正确做法：使用工具输出
 */
const correctApproach = {
    method: '使用 showboat exec 或类似工具',
    benefits: [
        '避免手动复制错误',
        '确保代码片段准确',
        '可重复执行验证',
        '减少幻觉风险'
    ],
    example: 'showboat exec sed -n "10,30p" src/utils.js'
};

/**
 * 错误做法：手动复制代码
 */
const wrongApproach = {
    method: '手动复制粘贴代码到文档',
    risks: [
        '可能复制错误',
        '代码可能过时',
        '无法验证准确性',
        '增加幻觉风险'
    ]
};
```

### 常用工具命令

```bash
# 查看文件内容
cat filename.py

# 查看文件部分内容
sed -n '10,30p' filename.py

# 搜索包含关键字的行
grep -n "functionName" filename.py

# 显示文件结构
head -50 filename.py

# 查找文件
find . -name "*.py" -type f

# 统计代码行数
wc -l filename.py
```
