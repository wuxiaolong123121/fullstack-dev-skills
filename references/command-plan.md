# /plan 规划命令 (Plan Command)

## 概述

`/plan` 命令用于功能规划与任务分解，将复杂需求拆分为可执行的子任务，生成详细的实施计划和工时估算。

## 命令语法

```
/plan <功能描述> [选项]
```

### 参数说明

| 参数 | 类型 | 必需 | 描述 |
|-----|------|------|------|
| `<功能描述>` | string | 是 | 要规划的功能或任务描述 |
| `--epic` | flag | 否 | 创建 Epic 级别的大型规划 |
| `--sprint` | flag | 否 | 创建 Sprint 级别的迭代规划 |
| `--tasks` | flag | 否 | 仅输出任务列表 |
| `--estimate` | flag | 否 | 包含工时估算 |
| `--dependencies` | flag | 否 | 分析任务依赖关系 |
| `--output` | string | 否 | 输出格式 (text/json/markdown) |

## 使用示例

### 基础用法
```
/plan 实现用户登录功能
```

### Epic 级别规划
```
/plan 构建电商订单系统 --epic --estimate
```

### Sprint 规划
```
/plan 本周开发任务 --sprint --dependencies
```

### 仅输出任务列表
```
/plan 重构支付模块 --tasks
```

```markdown
### 任务分解

├── Story-001: 用户注册
│   ├── TASK-001: 设计用户数据模型
│   ├── TASK-002: 实现注册 API
│   └── TASK-003: 编写注册测试
│
├── Story-002: 用户登录
│   ├── TASK-004: 实现登录 API
│   ├── TASK-005: 实现 JWT 认证
│   └── TASK-006: 编写登录测试
│
└── Story-003: 密码管理
    ├── TASK-007: 实现密码重置
    └── TASK-008: 实现密码修改

### 工时估算
| 任务 | 预估工时 | 优先级 |
|-----|---------|-------|
| TASK-001 | 2h | P0 |
| TASK-002 | 4h | P0 |
| TASK-003 | 2h | P0 |
| TASK-004 | 4h | P1 |
| TASK-005 | 3h | P1 |
| TASK-006 | 2h | P1 |
| TASK-007 | 3h | P2 |
| TASK-008 | 2h | P2 |
| **总计** | **22h** | - |

### 依赖关系
```
TASK-001 ──▶ TASK-002 ──▶ TASK-003
     │
     └──────▶ TASK-004 ──▶ TASK-005 ──▶ TASK-006
                                    │
                                    └──▶ TASK-007
                                    │
                                    └──▶ TASK-008
```

### 里程碑
- M1 (Day 1-2): TASK-001, 002, 003 - 用户注册完成
- M2 (Day 3-4): TASK-004, 005, 006 - 用户登录完成
- M3 (Day 5): TASK-007, 008 - 密码管理完成

### 风险评估
| 风险 | 可能性 | 影响 | 缓解措施 |
|-----|-------|------|---------|
| JWT 安全漏洞 | 中 | 高 | 使用成熟库，定期审计 |
| 第三方登录集成 | 高 | 中 | 预留额外时间 |
```

## 规划模板

### Epic 模板
```markdown
# Epic: [Epic 名称]

## 背景
[为什么需要这个 Epic]

## 目标
- [目标 1]
- [目标 2]

## 范围
### 包含
- [功能 1]
- [功能 2]

### 不包含
- [排除项 1]

## Stories
1. [Story 1]
2. [Story 2]

## 成功标准
- [ ] [标准 1]
- [ ] [标准 2]
```

```markdown
# Story: [Story 名称]

## 作为
[用户角色]

## 我想要

- [ ] [标准 1]
- [ ] [标准 2]

- [ ] [任务 1]
- [ ] [任务 2]
```


## 最佳实践

### 1. 需求描述
- 使用清晰、具体的语言
- 包含业务背景和目标
- 明确范围边界

### 2. 任务分解
- 任务粒度适中 (2-8 小时)
- 任务边界清晰
- 单一职责原则

### 3. 工时估算
- 基于历史数据
- 考虑风险缓冲
- 团队成员技能差异

### 4. 依赖管理
- 最小化依赖关系
- 识别关键路径
- 规划并行任务
