# instinct-status 命令参考

## 命令概述
`instinct-status` 命令用于查看已学习的本能（instincts）及其置信度分数，帮助用户了解系统的学习状态和知识积累情况。

## 命令语法
```bash
fullstack-dev-skills instinct-status [选项]
```

## 参数说明
| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `--filter` | 字符串 | 否 | 按类别或关键词过滤本能 |
| `--sort` | 字符串 | 否 | 排序方式，支持 `confidence`（按置信度）、`date`（按日期） |
| `--limit` | 数字 | 否 | 限制输出数量，默认 50 |
| `--output` | 字符串 | 否 | 输出格式，支持 `json` 和 `text`，默认 `text` |

## 工作原理
1. **数据收集**：从本地存储中读取已学习的本能数据
2. **数据处理**：根据过滤和排序参数处理数据
3. **置信度计算**：计算每个本能的置信度分数
4. **结果展示**：以清晰的格式展示本能及其置信度

## 常见问题

### Q: 什么是本能（instincts）？
A: 本能是系统从用户交互和执行任务中学习到的知识片段，包括代码模式、架构设计原则、调试技巧等。每个本能都有一个置信度分数，表示系统对该知识的确定程度。

### Q: 置信度分数是如何计算的？
A: 置信度分数基于多种因素计算，包括：
- 本能被使用的频率
- 本能的成功应用次数
- 相关本能的一致性
- 学习时间的远近

置信度分数范围为 0-1，越高表示系统对该知识越确定。

### Q: 如何提高本能的置信度？
A: 可以通过以下方式提高本能的置信度：
- 多次成功应用相关知识
- 提供明确的反馈（正面或负面）
- 重复类似的任务，强化相关知识
- 导入高质量的外部本能

### Q: 低置信度的本能会被系统丢弃吗？
A: 系统不会自动丢弃低置信度的本能，但会降低它们在决策中的权重。随着系统学习更多相关知识，低置信度的本能可能会被更新或合并到更准确的本能中。

### Q: 本能数据存储在哪里？
A: 本能数据存储在本地文件系统中，通常位于用户目录下的 `.trae/instincts` 文件夹中。这些数据是私有的，不会被共享或上传到任何服务器。
