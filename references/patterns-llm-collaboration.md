# LLM 协作范式参考

人机协作平衡理念与上下文管理艺术，用于高效利用 AI 编码代理。

## When to Activate
- 开始新的 AI 辅助开发会话时
- 遇到复杂任务需要上下文管理时
- 需要迭代修正生成代码时

## 核心理念：人机协作平衡

### LLM 角色定义
- **本质**："过度自信的结对编程助手"
- **价值**：加速重复性工作、探索性学习工具、知识辅助
- **边界**：开发者始终掌握最终决策权

### 价值放大三要素
1. **加速重复性工作**：生成样板代码、查找文档示例
2. **探索性学习工具**：快速原型验证，降低试错成本
3. **知识辅助**：解答代码库问题，缩短上下文切换时间

### 决策权划分
| 开发者负责 | LLM 负责 |
|-----------|---------|
| 系统架构设计 | 代码片段生成 |
| 安全性决策 | 文档查找 |
| 关键业务逻辑 | 测试用例编写 |
| 最终代码审查 | 简单 Bug 修复 |

## 上下文管理艺术

### 对话重置策略

```javascript
/**
 * 何时重置对话
 * @description 复杂任务中旧对话的残留上下文可能导致偏离目标
 */
const resetTriggers = {
    taskSwitch: '切换到完全不同的任务',
    contextPollution: '模型开始引用不相关信息',
    repeatedErrors: '连续多次修正失败',
    scopeCreep: '模型开始添加未请求的功能'
};
```

### 上下文注入技巧

#### 代码示例注入
```javascript
/**
 * 上下文注入示例
 * @description 先提供相似功能的代码，再要求修改
 */
// 步骤 1: 提供参考代码
const referenceCode = `
async function fetchUser(id) {
    const response = await fetch(\`/api/users/\${id}\`);
    return response.json();
}
`;

// 步骤 2: 要求基于参考修改
const prompt = `
参考上面的 fetchUser 函数，实现一个带重试机制的版本：
- 最多重试 3 次
- 每次重试间隔 1 秒
- 处理网络错误
`;
```

#### 分步引导策略
```javascript
/**
 * 分步引导流程
 * @description 先生成简单实现，再逐步添加功能
 */
const stepByStepGuide = {
    step1: '先实现基本的下载功能',
    step2: '添加错误处理',
    step3: '添加重试逻辑',
    step4: '添加进度显示',
    step5: '添加取消功能'
};
```

### 精准反馈示例

```javascript
/**
 * 精准反馈对比
 * @description 好的反馈 vs 不好的反馈
 */

// ❌ 不好的反馈
const badFeedback = '代码不工作，请修复';

// ✅ 好的反馈
const goodFeedback = `
第30行的 parse_response 函数未处理 JSONDecodeError。
当 API 返回非 JSON 格式的错误响应时，程序崩溃。

期望行为：
1. 捕获 JSONDecodeError
2. 记录原始响应内容
3. 返回有意义的错误信息
`;
```

## 沙箱化执行环境

### 推荐工具

| 工具 | 适用场景 | 安全特性 |
|------|----------|----------|
| ChatGPT Code Interpreter | 数据处理、Python 脚本 | 隔离执行、无网络访问 |
| Claude Artifacts | Web 应用预览 | 客户端渲染、无服务器访问 |
| 本地 Docker 容器 | 完整应用测试 | 网络隔离、资源限制 |
| 虚拟机 | 系统级测试 | 完全隔离 |

### 安全原则

```javascript
/**
 * 沙箱执行安全清单
 */
const sandboxChecklist = {
    never: [
        '在生产环境直接运行未审核代码',
        '执行包含敏感信息的代码',
        '运行需要管理员权限的代码'
    ],
    always: [
        '先在隔离环境测试',
        '审查代码中的网络请求',
        '检查文件系统操作',
        '验证依赖来源'
    ]
};
```

## 最佳实践清单

### 会话开始时
- [ ] 明确当前任务目标
- [ ] 评估是否需要重置对话
- [ ] 准备必要的上下文（代码示例、文档）

### 迭代过程中
- [ ] 记录每次修改和原因
- [ ] 保持反馈精准具体
- [ ] 定期要求模型解释关键逻辑

### 会话结束时
- [ ] 运行完整测试套件
- [ ] 审查所有生成代码
- [ ] 更新文档和注释
