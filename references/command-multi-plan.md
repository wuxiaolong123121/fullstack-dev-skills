# multi-plan 命令参考

## 命令概述
`multi-plan` 命令用于多模型协作规划，通过协调多个不同类型的模型来共同完成复杂的规划任务。

## 命令语法
```bash
fullstack-dev-skills multi-plan [选项] --task "任务描述"
```

## 参数说明
| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `--task` | 字符串 | 是 | 要规划的任务描述，详细说明任务目标和要求 |
| `--models` | 字符串数组 | 否 | 指定要使用的模型列表，默认为系统推荐的模型组合 |
| `--timeout` | 数字 | 否 | 任务超时时间（秒），默认 300 |
| `--output` | 字符串 | 否 | 输出格式，支持 `json` 和 `text`，默认 `text` |

## 工作原理
1. **任务分解**：将复杂任务分解为多个子任务
2. **模型分配**：根据子任务的性质分配给最适合的模型
3. **并行处理**：多个模型同时处理各自的子任务
4. **结果整合**：将各模型的结果整合为统一的规划方案
5. **一致性检查**：确保规划方案的内部一致性和可行性

## 常见问题

### Q: multi-plan 命令与普通规划命令有什么区别？
A: `multi-plan` 命令通过协调多个不同类型的模型来完成规划任务，可以整合不同模型的优势，提供更全面、更深入的规划方案。

### Q: 如何选择适合的模型组合？
A: 系统会根据任务类型自动推荐合适的模型组合。对于技术类任务，建议包含擅长技术分析的模型；对于创意类任务，建议包含擅长创意生成的模型。

### Q: 任务描述应该多详细？
A: 任务描述越详细，规划结果越准确。建议包含任务目标、具体要求、约束条件、预期成果等信息。

### Q: 如何处理规划结果不符合预期的情况？
A: 可以尝试以下方法：
1. 提供更详细的任务描述
2. 指定更适合的模型组合
3. 调整任务的范围和目标
4. 多次运行命令，比较不同的规划结果
