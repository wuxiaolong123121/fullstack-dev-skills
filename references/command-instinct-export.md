# instinct-export 命令参考

## 命令概述
`instinct-export` 命令用于将本地学习的本能（instincts）导出到外部文件，帮助用户备份学习成果、在不同系统之间迁移知识，或与其他用户共享学习经验。

## 命令语法
```bash
fullstack-dev-skills instinct-export [选项] --file <文件路径>
```

## 参数说明
| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `--file` | 字符串 | 是 | 导出文件的路径，将以 JSON 格式保存 |
| `--filter` | 字符串 | 否 | 按类别或关键词过滤要导出的本能 |
| `--min-confidence` | 数字 | 否 | 最小置信度阈值，低于此值的本能将被跳过，默认 0 |
| `--limit` | 数字 | 否 | 限制导出数量，默认无限制 |
| `--format` | 字符串 | 否 | 导出格式，支持 `json`（默认）和 `jsonl`（每行一个 JSON 对象） |
| `--include-usage` | 布尔值 | 否 | 是否包含使用统计信息，默认 `true` |

## 工作原理
1. **数据收集**：从本地存储中读取本能数据
2. **过滤处理**：根据过滤参数和置信度阈值筛选本能
3. **数据格式化**：将筛选后的本能转换为指定格式
4. **文件写入**：将格式化的数据写入指定文件
5. **结果汇总**：生成导出结果报告

## 常见问题

### Q: 导出的本能文件包含哪些信息？
A: 导出的本能文件包含以下信息：
- 本能的基本信息（ID、类别、描述等）
- 本能的置信度分数
- 本能的使用示例和上下文信息
- 本能的使用统计（如果启用了 `--include-usage`）
- 导出元数据（版本、导出时间等）

### Q: 如何选择导出格式？
A: 选择导出格式时考虑以下因素：
- **JSON 格式**：适合大多数用途，易于阅读和编辑，适合导入到其他系统
- **JSONL 格式**：适合处理大型数据集，便于使用命令行工具（如 jq）进行处理，适合机器学习或数据分析任务

### Q: 导出大量本能会影响系统性能吗？
A: 导出大量本能可能会暂时影响系统性能，但导出过程是只读操作，不会修改任何数据，也不会长期影响系统性能。

### Q: 如何确保导出的本能质量？
A: 您可以使用以下参数来控制导出的本能质量：
- `--min-confidence`：设置最小置信度阈值
- `--filter`：只导出特定类别的本能
- `--limit`：限制导出数量，优先导出质量较高的本能

### Q: 导出失败的原因有哪些？
A: 导出失败的常见原因包括：
- 文件路径无效或无法访问
- 磁盘空间不足
- 文件权限问题
- 系统资源不足

### Q: 如何与其他用户共享导出的本能？
A: 与其他用户共享导出的本能时，建议：
1. 使用 `--min-confidence` 参数过滤低质量的本能
2. 使用 `--filter` 参数只共享特定类别的本能
3. 确保文件格式正确，便于其他用户导入
4. 提供简要说明，帮助其他用户了解导出的本能内容

### Q: 定期导出本能有什么好处？
A: 定期导出本能的好处包括：
- **备份**：防止数据丢失
- **迁移**：在不同系统之间移动学习成果
- **共享**：与团队成员或社区共享学习经验
- **分析**：对学习模式和知识积累进行分析
- **版本控制**：将导出文件纳入版本控制系统，跟踪学习进展

### Q: 导出的本能文件大小有限制吗？
A: 导出文件的大小取决于本能的数量和详细程度。对于非常大的本能集合，建议：
- 使用 `--filter` 和 `--min-confidence` 参数减少导出数量
- 使用 `--format jsonl` 格式，便于处理大型文件
- 考虑分批导出不同类别的本能

### Q: 如何查看导出文件的内容？
A: 您可以使用以下方法查看导出文件的内容：
- 文本编辑器：打开 JSON 文件查看
- 命令行工具：使用 `cat`、`less` 或 `jq` 等工具查看
- 导入到系统：使用 `instinct-import` 命令将文件导入到系统中查看
